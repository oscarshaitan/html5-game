TECHNICAL REFERENCE

Neon Defense Engine Architecture & API Reference

STATE LOGIC SPAWNING UI AUDIO RENDERING

01. ENGINE STATE

resetGameLogic()
Initializes or wipes all game variables (money, lives, wave, towers) to start state.
saveGame()
Serializes the current state into localStorage under the key 'neonDefenseSave'.
loadGame()
Parses localStorage and restores state, including complex objects like towers and mutant definitions.

02. CORE LOGIC

gameLoop(timestamp)
The primary RAF-driven loop. Orchestrates update() and draw().
update()
Handles frame-by-frame updates for entities, physics, and timers. Runs only if game is playing and not paused.
isValidPlacement(x, y, config)
Checks grid collision against paths, UI bounds, and existing towers. Returns snap coordinates. Update: Now blocks selection if targeting a cell occupied by a Rift Path.

03. ENEMY & PATHING

calculatePath()
Generates the initial maze path using a randomized BFS algorithm from edge to center.
spawnEnemy()
Pops the next enemy from spawnQueue, scales HP by wave, and assigns a random active path.
generateMutation()
(Internal) Logic hook for creating mutated enemy variants every 20 waves.
generateNewPath()
Generates an independent path from an orbital shell to the core or a neighbor.
  • Spatial Zoning: Rifts fill concentric shells ($6 \pm 3n$ units) sequentially.
  • Shell Capacity: Limits paths per zone using the $2n^2$ model (Orbit 1: 2, Orbit 2: 8, etc.).
  • Connectivity: Direct linkage to core follows $P = 1/2^{n-1}$ probability curve. Failing this, it seeks the absolute closest neighbor segment for a merge.
findPathOnGrid(start, end, obstacles)
A* pathfinder with dynamic weighting.
  • Crossing Penalties: Existing rifts are treated as high-resistance terrain (cost: 100).
  • Turn Penalties: Penalty of 5 for direction shifts to favor straight segments.

04. COMMANDER ABILITIES

activateAbility(type)
Toggles the targeting state for a specific ability (emp, overclock). Blocks tower building while active. Refuses activation if Energy is insufficient or ability is on cooldown.
useAbility(type, target)
Executes the ability effect. Deducts Energy, sets the maxCooldown, and applies logic:
  • emp: Flags enemies in radius with frozen: true and sets frozenTimer.
  • overclock: Flags a tower with overclocked: true and sets overclockTimer.

05. USER INTERFACE

loadGame()
Parses localStorage and restores state, including persistent Core Tower levels and localized Rift (Path) data.
updateUI()
Syncs HTML elements (money, lives, timers) with the internal engine state.
updateSelectionUI()
Dynamically renders and positions the floating upgrade/sell panel near the selected entity.
getRepairCost()
Returns the current price for life restoration based on lives exceeding parity.
updateWavePanel()
Calculates incoming wave statistics (enemy count, sector evolution odds, active mutations) and populates the Wave Intelligence modal with live-syncing data.

05. AUDIO ENGINE

AudioEngine.updateMusic()
A multi-track sequencer that plays 16-step patterns. Shifts between 'Normal' and 'Threat' themes based on enemy presence.
AudioEngine.playArp(freq, type, vol, duration)
Generates a rapid arpeggiated sequence (8-bit style) used for high-threat melodies.
AudioEngine.setVolume(type, val)
Adjusts gain nodes for 'music' or 'sfx' and persists settings to localStorage.
AudioEngine.toggleMute()
Toggles the master gain node between 0 and 1 with a smooth 100ms ramp. Synchronizes UI mute buttons.
AudioEngine.init()
Initializes the Web Audio API context. Resumes context if suspended (browser security requirement).

06. RENDERING PIPELINE

draw()
Total canvas redraw. Applies camera transforms (pan/zoom) before drawing entities.
drawTowerOne(type, x, y, color)
Graphic primitive for drawing specific tower shapes (Square, Circle, Diamond).
drawLevelPips(tower)
Draws visual markers at the base of towers to represent upgrade level.
