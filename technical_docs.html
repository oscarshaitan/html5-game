TECHNICAL REFERENCE

Neon Defense Engine Architecture & API Reference

STATE LOGIC SPAWNING UI RENDERING

01. ENGINE STATE

resetGameLogic()
Initializes or wipes all game variables (money, lives, wave, towers) to start state.
saveGame()
Serializes the current state into localStorage under the key 'neonDefenseSave'.
loadGame()
Parses localStorage and restores state, including complex objects like towers and mutant definitions.

02. CORE LOGIC

gameLoop(timestamp)
The primary RAF-driven loop. Orchestrates update() and draw().
update()
Handles frame-by-frame updates for entities, physics, and timers. Runs only if game is playing and not paused.
isValidPlacement(x, y, config)
Checks grid collision against paths, UI bounds, and existing towers. Returns snap coordinates.

03. ENEMY & PATHING

calculatePath()
Generates the initial maze path using a randomized BFS algorithm from edge to center.
spawnEnemy()
Pops the next enemy from spawnQueue, scales HP by wave, and assigns a random active path.
generateMutation()
(Internal) Logic hook for creating mutated enemy variants every 20 waves.

04. USER INTERFACE

updateUI()
Syncs HTML elements (money, lives, timers) with the internal engine state.
updateSelectionUI()
Dynamically renders and positions the floating upgrade/sell panel near the selected entity.
getRepairCost()
Returns the current price for life restoration based on lives exceeding parity.

05. RENDERING PIPELINE

draw()
Total canvas redraw. Applies camera transforms (pan/zoom) before drawing entities.
drawTowerOne(type, x, y, color)
Graphic primitive for drawing specific tower shapes (Square, Circle, Diamond).
drawLevelPips(tower)
Draws visual markers at the base of towers to represent upgrade level.
